# 📄 Автозаполнение документов Word

Streamlit-приложение для массового заполнения Word-шаблонов данными сотрудников.

## Установка

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Как использовать

### 1. Подготовьте шаблоны (.docx)

Расставьте плейсхолдеры в документе — они должны совпадать с заголовками колонок в таблице.

**Поддерживаемые форматы плейсхолдеров** (выбираются в настройках):
- `{{ФИО}}` — двойные фигурные скобки (по умолчанию)
- `{ФИО}` — одинарные фигурные скобки
- `[[ФИО]]` — квадратные скобки
- `<<ФИО>>` — угловые скобки

**Пример текста в шаблоне:**
```
Настоящий договор заключён с {{ФИО}}, паспорт серия {{ПАСПОРТ_СЕРИЯ}} № {{ПАСПОРТ_НОМЕР}},
проживающим по адресу {{АДРЕС}}. Контактный email: {{EMAIL}}.
```

> ⚠️ **Важно:** плейсхолдер должен быть написан целиком в одном месте без разрывов.
> Если Word «разбил» его при вводе — удалите и напечатайте заново.

### 2. Подготовьте таблицу (.xlsx или .csv)

| ФИО | EMAIL | ПАСПОРТ_СЕРИЯ | ПАСПОРТ_НОМЕР | АДРЕС |
|-----|-------|---------------|---------------|-------|
| Иванов Иван Иванович | ivanov@mail.ru | 4510 | 123456 | г. Москва... |
| Петров Пётр Петрович | petrov@mail.ru | 4511 | 654321 | г. СПб... |

- Заголовки колонок = имена плейсхолдеров в шаблонах
- Одна строка = один сотрудник

### 3. Запустите генерацию

После загрузки файлов нажмите **«Создать ZIP-архив»** и скачайте результат.

### Структура ZIP-архива

```
documents.zip
├── Иванов_Иван_Иванович/
│   ├── договор.docx
│   ├── анкета.docx
│   └── приказ.docx
├── Петров_Пётр_Петрович/
│   ├── договор.docx
│   └── ...
```

## Технические детали

- Замена происходит на уровне XML внутри `.docx` (ZIP-архив с XML-файлами)
- Перед заменой соседние runs с одинаковым форматированием склеиваются — это решает проблему, когда Word разбивает `{{ФИО}}` на несколько фрагментов
- Все остальные части документа (стили, изображения, таблицы, колонтитулы) остаются нетронутыми
- Заменяются плейсхолдеры в основном тексте, колонтитулах и сносках
